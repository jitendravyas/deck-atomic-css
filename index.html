Straightforward

simple, obvious classes are easy to use and internalize

bonus: you know what components will look like just by reading the markup

img class="circle border" vs. img class="profile-pic"

Reusable

classes that describe content can only be used where that content exists

classes that describes a visual pattern can be used in any context

Encapsulation is your best friend

flat list of classes means no more worrying about global scope, specificity wars, overrides, descendant/contextual selectors

by encapsulating work in an individual block of html, you can refactor with the confidence you aren't breaking anything else

bonus: this means components are 100% portable

Beginner friendly

writing efficient and robust CSS is difficult

reusing small, existing classes simplifies the learning curve and democratizes the design process üá∫üá∏

Brings engineers and designers together

the DOM becomes a design tool

designers can jump in the code & engineers can influence design decisions and everyone is now speaking a shared language
